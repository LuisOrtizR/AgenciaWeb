Crea cuenta → devuelve tokenAcceso + tokenRefresco
POST
/autenticacion/login
Login → devuelve tokens
POST
/autenticacion/refrescar
Renueva el token (expira en 8h)
📌 En tu frontend:
1. Guarda tokenAcceso en localStorage
2. En cada request protegido agrega:
Authorization: Bearer {tokenAcceso}
3. Si recibes 401 → usa tokenRefresco para renovar

📦 Las 5 entidades principales

🛒 SERVICIO — El catálogo de lo que ofreces
Son los productos/paquetes que vendes (Ej: Landing Page, E-commerce). Tienen rango de precio en COP, tiempo de entrega y características. Son públicos. Los precios se muestran también en USD con tasa en tiempo real.
nombreslugprecioDesde/HastasemanasEntregacaracteristicas[]
🖼️ PROYECTO — Tu portafolio
Trabajos realizados que muestras al público. Cada proyecto tiene stack tecnológico, URL en vivo, imagen en Cloudinary y puede marcarse como "destacado" para aparecer en la home.
tituloslugstackTecnico[]urlEnVivoimagendestacado
📋 PROSPECTO — Lead / Contacto del formulario
Cualquier persona que llena el formulario. No necesita estar registrada. El admin los gestiona como un CRM. Si el correo coincide con un usuario registrado, se vinculan automáticamente.
nombrecorreotelefonotipoServiciopresupuestofuente
Pipeline de estados:
NUEVO → CONTACTADO → CONVERTIDO | PERDIDO
Al crear cotización: NUEVO → CONTACTADO automático
Al aceptar cotización: → CONVERTIDO automático
Al rechazar cotización: → PERDIDO automático
💰 COTIZACIÓN — La propuesta económica
El admin crea una cotización para un prospecto, vinculada a un servicio. Al marcarla como ENVIADA, el sistema envía un correo automático. El cliente la acepta o rechaza desde su panel.
Estados:
PENDIENTE → ENVIADA → ACEPTADA | RECHAZADA
⭐ TESTIMONIO — Reviews de clientes
Los clientes los envían públicamente. Empiezan ocultos hasta que el admin los aprueba. El campo sitioWeb es un honeypot anti-bots (debe ir vacío). Solo los visibles con calificación ≥4 aparecen en el sitio.
nombreClientecontenidocalificacion (1-5)visible (admin aprueba)
